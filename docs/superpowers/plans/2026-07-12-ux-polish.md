# UX Polish (Track 2) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add synthesized sound effects (correct/incorrect/island-complete/creature-unlocked) and profile deletion to Word Islands.

**Architecture:** A new `src/sound.js` module wraps the Web Audio API the same way `src/speech.js` wraps the Web Speech API — defensive, no-op when unavailable, zero asset files. A `soundOn` boolean joins `uiLang` as top-level app state (device setting, not per-profile), threaded down as a prop through `App → Island → {QuickQuiz,TapTheRightOne,MemoryMatch}`. Profile deletion is a new pure `deleteProfile` function in `progress.js`, wired into a new delete button on each `ProfilePicker` card behind a `window.confirm`.

**Tech Stack:** React 18, Vite, Vitest (default `node` test environment — no jsdom, no component rendering in tests, matching this project's existing test suite).

## Global Constraints

- Sound effects: Web Audio API synthesized tones only, no audio asset files (spec §1).
- `soundOn` defaults to `true`, is a device-level setting (top-level app state, same tier as `uiLang`), not per-profile (spec §1).
- Every `sound.js` function must be a safe no-op when disabled or when the Web Audio API is unavailable — never throws (spec §1).
- Profile deletion uses a simple `window.confirm` dialog naming the profile, not type-to-confirm (spec §2).
- Deleting the active profile clears `activeProfileId` to `null` and falls back to the existing profiles screen — no new screen-routing logic (spec §2).
- No new profile-switching UI — the existing header chip → picker flow already covers switching (spec, non-goals).
- Every English string added to `i18n.js` needs a Hebrew counterpart — enforced by the existing `i18n.test.js` parity test.
- Follow existing code style: pure functions in `progress.js`/`sound.js` take/return plain data, no classes; components take plain props, no context/redux.

```js
// Thin wrapper over the Web Audio API for short UI sound effects.
// No audio files — every sound is a synthesized tone. Every exported
// function is a no-op when sound is disabled or the API is
// unavailable, so the game never breaks just because audio can't play.

let ctx = null;

function getContext() {
  if (ctx) return ctx;
  const Ctor = typeof window !== 'undefined' && (window.AudioContext || window.webkitAudioContext);
  if (!Ctor) return null;
  ctx = new Ctor();
  return ctx;
}

function tone(context, freq, startOffset, duration, type = 'sine', peakGain = 0.2) {
  const osc = context.createOscillator();
  const gain = context.createGain();
  osc.type = type;
  osc.frequency.value = freq;
  const start = context.currentTime + startOffset;
  const end = start + duration;
  gain.gain.setValueAtTime(0, start);
  gain.gain.linearRampToValueAtTime(peakGain, start + 0.01);
  gain.gain.linearRampToValueAtTime(0, end);
  osc.connect(gain);
  gain.connect(context.destination);
  osc.start(start);
  osc.stop(end);
}

export function playCorrect(enabled) {
  if (!enabled) return;
  const context = getContext();
  if (!context) return;
  tone(context, 659.25, 0, 0.12); // E5
  tone(context, 987.77, 0.1, 0.15); // B5
}

export function playIncorrect(enabled) {
  if (!enabled) return;
  const context = getContext();
  if (!context) return;
  tone(context, 180, 0, 0.25, 'sawtooth', 0.15);
}

export function playStars(enabled) {
  if (!enabled) return;
  const context = getContext();
  if (!context) return;
  tone(context, 523.25, 0, 0.15); // C5
  tone(context, 659.25, 0.13, 0.15); // E5
  tone(context, 783.99, 0.26, 0.25); // G5
}

export function playCreatureUnlocked(enabled) {
  if (!enabled) return;
  const context = getContext();
  if (!context) return;
  tone(context, 523.25, 0, 0.12);
  tone(context, 659.25, 0.1, 0.12);
  tone(context, 783.99, 0.2, 0.12);
  tone(context, 1046.5, 0.3, 0.3);
}
```
